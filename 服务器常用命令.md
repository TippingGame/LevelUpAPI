# LevelUpAPI 服务器安全更新与构建命令

服务器项目目录：

```text
/opt/LevelUpAPI
```

当前部署使用：

```text
deploy/docker-compose.local.yml
deploy/docker-compose.local-build.yml
```

## 首次准备：4GB 服务器创建 2GB Swap

先检查：

```bash
free -h
```

如果显示 `Swap: 0B`，执行一次：

```bash
[ -f /swapfile ] || sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

grep -q '^/swapfile ' /etc/fstab || \
  echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-levelup-swap.conf
sudo sysctl --system
free -h
```

Swap 用于吸收构建时的短时内存峰值，防止网站和 SSH 因 OOM 一起失联。

## 标准更新流程

### 1. 拉取代码

```bash
cd /opt/LevelUpAPI
git pull origin main
git log -1 --oneline
cat backend/cmd/server/VERSION
```

### 2. 保存当前运行镜像

```bash
docker image tag \
  "$(docker inspect -f '{{.Image}}' sub2api)" \
  levelupapi-local:rollback-before-update
```

### 3. 单独进行低资源构建

```bash
cd /opt/LevelUpAPI/deploy

COMPOSE_PROGRESS=plain docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  build sub2api
```

构建覆盖文件默认启用以下保护：

- 构建进程低优先级；
- 前端和 Go 编译使用单线程；
- Node 堆上限 1792MB；
- Go 编译软内存限制 768MiB；
- 生产前端只执行一次类型检查。

构建期间旧的 `sub2api` 容器继续提供服务。不要先执行 `docker compose down`。

### 4. 构建成功后替换应用容器

```bash
docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  up -d --no-build --no-deps sub2api

sleep 30
```

`--no-deps` 会保留正在运行的 PostgreSQL 和 Redis，不因应用更新而重启它们。

### 5. 验证

```bash
docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  ps

curl -fsS http://127.0.0.1:8080/health
curl -I http://127.0.0.1:8080/login

sudo nginx -t && sudo systemctl reload nginx
curl -I https://levelup.mom/login

docker stats --no-stream sub2api sub2api-postgres sub2api-redis
free -h
df -h
```

## 构建期间服务器变卡

如果 SSH 或网站开始明显变卡，立即在构建终端按：

```text
Ctrl+C
```

中断构建不会停止旧应用。随后确保已有容器继续运行：

```bash
cd /opt/LevelUpAPI/deploy

docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  up -d --no-build
```

不要执行：

```bash
docker compose down
docker system prune
docker builder prune -a
docker volume prune
```

## 新版本启动失败时回滚

```bash
docker image tag \
  levelupapi-local:rollback-before-update \
  levelupapi-local:latest

cd /opt/LevelUpAPI/deploy

docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  up -d --force-recreate --no-build --no-deps sub2api

sleep 30
curl -fsS http://127.0.0.1:8080/health
```

## 查看应用日志

```bash
cd /opt/LevelUpAPI/deploy

docker compose \
  -f docker-compose.local.yml \
  -f docker-compose.local-build.yml \
  logs -f --tail=100 sub2api
```

退出日志跟随使用 `Ctrl+C`，不会停止容器。
